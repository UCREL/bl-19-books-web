You can query for both tokens and Part Of Speech (POS) tags, of which these are represented by the token and pos columns respectively. For a list of all POS tags and their description see the POS tagset section below.

We used the Spacy English (medium) model to generate the tokens and POS tags that can be queried through the LexiDB database.

Basics and QBE expressions

The query syntax in LexiDB is based around QBE (Query by example). An example of a simple QBE object would be;

{"token": "test"}

This would query for the string test in the token column.

Regular expressions can be used instead of string values;

{"token": "test.*"}

This query would return results for all tokens that match the regex test.* i.e. test, tested, testing, etc.

Additional columns can be queried within a QBE object, expressed as a comma-seperated list of key-value pairs. For example, Part of Speech can be included;

{"token": "test.*", "pos":"V.*"}

The above query would find all instances of tokens starting with test that are tagged with a POS beginning V (a verb form). All columns can be queried using regular expressions.

Note in a single QBE object all key-value pairs specified will be marched against in the query. LexiDB's QBE syntax is essentially JavaScript Object Notation (JSON).

Sequences and token stream regular expressions

Sequences of tokens can be queried for by expressing a series of QBE objects;

{"token": "test"}{"token": "for"}{"token": "a"}

This query would search for the sequence test for a.

Within a series of QBE objects regular expression special characters may be used;

{"token": "test"}{"token": "for"}?{"token": "a"}

Above the ? after the second QBE object specifies that this token is optional and so the query would return results test for a and test a.

Regular expressions

LexiDB makes use of dk.brics automata engine for regular expression matching both on string values within columns and token values on sequences. Any special regular expression characters not supported by this engine are not currently supported by LexiDB.

Advanced usage

QBE objects can be back-referenced within a query. For example, the following query searches for the term run followed by a verb or a verb followed by the term run;

{"token": "run"}{"pos": "V.*"}|ba

In this case the first QBE object {"token": "run"} can be back-referenced using a and the second object {"pos": "V.*"} cab be back-referenced using b. The same query could be written more verbosely as;

{"token": "run"}{"pos": "V.*"}|{"pos": "V.*"}{"token": "run"}

Note for efficiency it is always recommended using back-references to prevent additional index lookups.

Examples

Search for the term war within 5 tokens of a word that has been tagged as an adjective JJ.*;

{"token": "war"}.{0,5}{"pos": "JJ.*"}|b.{0,5}a

Here as .{0,5} is out of the context of a QBE object it means any token up to 5 times rather than any character.
